## How to Submit an Error Report

**Via Telegram bot (admin):**
1. Admin menu → 🛡️ Dev → 📋 Report Error
2. Enter error name, then send text descriptions, voice notes, and/or screenshots
3. Tap ✅ Send — report is saved locally AND committed to this git repo

**Automatic log observer:**
- The bot monitors its own log file for ERROR/CRITICAL entries
- When ≥3 errors occur within 5 minutes, an auto-report is created here
- Set `GIT_ERRORS_DIR=<path-to-errors/>` in `bot.env` to enable git integration

## How to Resolve an Error (Developer Workflow)

Use the `/taris-error-review` Copilot skill:
```
/taris-error-review
```

Or manually:
1. Read `errors/<folder>/description.md` and `log_excerpt.txt`
2. Find the root cause in the code
3. Fix the bug + add a regression test (see `doc/lessons-learned.md` protocol)
4. Update `manifest.json`: set `"status": "resolved"`, add `"resolved_commit": "<sha>"`
5. Commit: `git add errors/<folder>/manifest.json && git commit -m "fix: resolve <error-name>"`
