## Problem

Currently, all new Telegram users who contact the bot are placed in `pending` state — they see only a "waiting for approval" message with zero functionality. This creates a poor first impression and blocks demo/onboarding/consultation scenarios.

**Goals:**
1. Allow a "Guest" mode where new users interact meaningfully before full approval
2. Allow a "User" mode where registration automatically grants full access (no admin approval required)
3. Enable guests to request consultations / book meetings with experts
4. Use admin-uploaded shared documents as knowledge base for guest chat

## Concept

### Registration Modes (configurable)

| Mode | Constant | On `/start` | Use case |
|---|---|---|---|
| **Pending** (default) | — | Admin must approve each request | Private/family bot |
| **Auto-Guest** | `AUTO_GUEST_ENABLED=1` | Immediate Guest access | Demo / public bot |
| **Auto-User** | `AUTO_USER_ENABLED=1` | Immediate full User access | Open community / intranet |

### Guest vs. User Access

| Feature | Pending | Guest | User |
|---|---|---|---|
| Chat (LLM) | ❌ | ✅ rate-limited, shared-docs RAG only | ✅ full, personal RAG |
| Notes | ❌ | ❌ | ✅ |
| Calendar (own) | ❌ | ❌ | ✅ |
| Request meeting / invite expert | ❌ | ✅ ⏳ | ✅ |
| View expert free time slots | ❌ | ✅ ⏳ | ✅ |
| Confirm meeting invitations | ❌ | ❌ | ✅ |
| Contacts | ❌ | ❌ | ✅ |
| Documents menu | ❌ | ❌ | ✅ |
| Upload documents | ❌ | ❌ | ✅ |
| Shared docs as chat knowledge | ❌ | ✅ (RAG context injected) | ✅ |
| Digest | ❌ | ❌ (needs e-mail account) | ✅ |
| Voice | ❌ | ❌ | ✅ |
| Web UI access | ❌ | ⏳ request form only | ✅ |
| Profile (view) | ❌ | ✅ | ✅ |
| Help | ✅ | ✅ | ✅ |
| Request full access | ✅ | ✅ | — |

## Guest Menu (Telegram)

Guest sees a simplified main menu:
- 💬 Chat
- 📅 Request Meeting (⏳)
- 👤 Profile
- ❓ Help
- 🔓 Request Full Access

All other callbacks (notes, calendar, contacts, documents, digest, agents, admin) → friendly redirect:  
*"This feature is available to registered users. Request full access to use it → [🔓 Request Access]"*

## Shared Documents as Guest Knowledge Base

Admin uploads "public" / "shared" documents (e.g., product info, FAQ, services catalogue) that are available to all users including guests.

**RAG injection for guests:** `_docs_rag_context(chat_id)` already retrieves from shared store — for guests, restrict to `chat_id = SHARED_DOCS_OWNER` (configurable, default: admin chat_id).

## Meeting Request / Calendar Booking (Guest ↔ Expert)

This feature enables guests to book consultations with users/experts via the bot.

### Flow

```
Guest: "Request Meeting"
  → Bot asks: topic, preferred date range, duration
  → Bot queries expert's free slots (reads expert calendar, finds gaps)
  → Bot shows available slots to guest
  → Guest selects slot → creates a "pending invitation" calendar event
  → Expert (User/Admin) receives notification: "Guest {name} requests meeting on {date}"
  → Expert: Confirm ✅ / Decline ❌ / Propose alternative 🔄
  → Guest notified of expert's decision
```

### Data Model

New calendar event type: `type = "invitation"` with fields:
- `guest_chat_id` — requester
- `expert_chat_id` — requested expert
- `topic` — meeting subject
- `proposed_dt` — proposed datetime
- `status` — `pending_expert` | `confirmed` | `declined` | `rescheduled`

### Implementation Notes

- Free slot algorithm: reads expert calendar in requested range, returns gaps ≥ `MIN_MEETING_DURATION` (default: 30 min)
- Expert's calendar private events shown as "busy" (no details to guest)
- Confirmed invitation creates real calendar event on expert side
- Guest receives ICS-format summary (or just text) after confirmation

## Rate Limiting Design

**Storage:** `_guest_message_counts: dict[int, dict]` in `bot_state.py`  
Each entry: `{"daily": int, "hourly": int, "hour_ts": float, "day_ts": float}`

| Limit | Default | Config constant |
|---|---|---|
| Max messages / day | 20 | `GUEST_MSG_DAILY_LIMIT` |
| Max messages / hour | 5 | `GUEST_MSG_HOURLY_LIMIT` |
| Max tokens / response | 500 | `GUEST_MAX_TOKENS` |

When limit hit → friendly message with "Request Full Access" button.
