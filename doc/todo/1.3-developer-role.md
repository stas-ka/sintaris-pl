# Developer Role & Dev Menu

**Status:** 🔲 Planned  
**Depends on:** RBAC §1.1 (infrastructure done v2026.3.30)

---

## Concept

Developer users get a dedicated 🛠 Developer menu with a coding-assistant LLM chat pre-primed with the bot's source code context. They can propose code changes, apply them, and restart the bot — all from within Telegram.

---

## Admin vs Developer System Chat Permissions

| Capability | Admin | Developer |
|---|---|---|
| Read files, view logs, check status | ✅ | ✅ |
| Change config files (edit settings) | ✅ | ✅ |
| Restart / stop / start services | ❌ Blocked | ✅ Confirm gate |
| Write / patch source code | ❌ Blocked | ✅ Confirm gate |
| Install packages, run `pip3` / `git` | ❌ Blocked | ✅ Confirm gate |
| Deploy files (`pscp`, `mv`) | ❌ Blocked | ✅ Confirm gate |
| Describe feature → agent implements | ❌ Blocked | ✅ Dev Chat mode |

Allowlists defined in `ADMIN_ALLOWED_CMDS` / `DEVELOPER_ALLOWED_CMDS` in `bot_security.py`.

---

## Dev Menu Buttons

| Button | Action |
|---|---|
| 💬 Dev Chat | LLM chat with source context injected (`bot_config`, `bot_handlers`, `dev-patterns.md`) |
| 🔄 Restart Bot | `systemctl restart picoclaw-telegram` — confirm gate |
| 📋 View Log | Last 30 lines of `telegram_bot.log` |
| 🐛 Last Error | Last ERROR/EXCEPTION line from journal |
| 📂 File List | `~/.picoclaw/*.py` with sizes + mtimes |
