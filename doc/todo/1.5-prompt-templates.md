### Existing: How System Prompts Are Built

**Multi-turn chat** (`_build_system_message()` in `bot_access.py:384`):
```python
SECURITY_PREAMBLE + _bot_config_block() + personal_ctx + memory_note + lang_instr
```
- `SECURITY_PREAMBLE` = `fmt_prompt(PROMPTS["security_preamble"], bot_name=BOT_NAME)`
- `_bot_config_block()` = hardcoded f-string (version, LLM, STT, TTS, capabilities)
- `personal_ctx` = calendar + notes + contacts (same for all roles — this is the gap)
- `lang_instr` = `PROMPTS["lang_instructions"][lang]`

**Single-turn** (`_with_lang()` in `bot_access.py:373`): similar, no personal_ctx.

**Voice** (`_voice_user_turn_content()` in `bot_access.py:428`): adds `stt_hints` when `[?word]` present.

## Problem Statement

The current `_build_system_message()` is role-blind: every user (guest, user, admin) receives identical personal context injection. This causes:
- Guests get calendar/notes context blocks (empty for them, but wastes tokens and sets wrong expectations)
- Conversation style/tone cannot differ per role
- `_bot_config_block()` capabilities list is hardcoded — guests see capabilities they can't use

---

## Goal

Extend `prompts.json` with **role-keyed system prompt templates**, rendered by the existing `fmt_prompt()` mechanism. No new loader module needed.

---

## Design

### Storage: New Section in `src/prompts.json`

```json
{
  "role_system_prompts": {
    "user":  "<template string with {placeholders}>",
    "guest": "<template string>",
    "admin": "<template string>",
    "voice": "<template string>"
  },
  "role_capabilities": {
    "user":  "<capabilities block for regular users>",
    "guest": "<capabilities block for guests — no docs, no voice, no digest>",
    "admin": "<capabilities block — full>"
  },
  "role_styles": {
    "guest": "You are speaking with a guest. Be welcoming and concise. Guide the user toward requesting full access to unlock more features.",
    "user":  "You are a personal assistant. Be professional, helpful, and context-aware.",
    "admin": "You are assisting a system administrator. Be efficient and technically precise."
  }
}
```

Templates use the **same `{identifier}` placeholder format** as all existing prompts.

### Template Variables

| Variable | Description | Guest | User | Admin | Voice |
|---|---|---|---|---|---|
| `{security_preamble}` | `PROMPTS["security_preamble"]` rendered | ✅ | ✅ | ✅ | ✅ |
| `{bot_name}` | `BOT_NAME` | ✅ | ✅ | ✅ | ✅ |
| `{user_name}` | User display name | ✅ | ✅ | ✅ | ✅ |
| `{lang_instruction}` | `PROMPTS["lang_instructions"][lang]` | ✅ | ✅ | ✅ | ✅ |
| `{bot_capabilities}` | `PROMPTS["role_capabilities"][role]` | ✅ (limited) | ✅ (full) | ✅ (full) | ✅ |
| `{style_guide}` | `PROMPTS["role_styles"][role]` | ✅ | ✅ | ✅ | ❌ |
| `{calendar_summary}` | `_calendar_context(chat_id)` output | ❌ | ✅ | ✅ | ✅ |
| `{notes_summary}` | `_notes_context(chat_id)` output | ❌ | ✅ | ✅ | ❌ |
| `{contacts_summary}` | `_contacts_context(chat_id)` output | ❌ | ✅ | ✅ | ❌ |
| `{memory_note}` | Memory coherence note | ❌ | ✅ | ✅ | ❌ |

### Role → Template Key Mapping

| Role | `role_system_prompts` key | `role_capabilities` key |
|---|---|---|
| `guest` | `guest` | `guest` |
| `user` | `user` | `user` |
| `advanced` | `user` | `user` |
| `admin` | `admin` | `admin` |
| `developer` | `admin` | `admin` |
| `voice` (any role in voice mode) | `voice` | `user` |

## Architecture

### No New Module Needed

All rendering uses existing `fmt_prompt()` from `src/core/bot_prompts.py`.

### New Helper: `_get_prompt_role_key(chat_id) -> str`

Add to `src/telegram/bot_access.py`:
```python
def _get_prompt_role_key(chat_id: int) -> str:
    """Map chat_id to a prompts.json role key."""
    if _is_admin(chat_id) or _is_developer(chat_id): return "admin"
    if _is_limited_guest(chat_id): return "guest"
    return "user"  # user, advanced — same template
```

### Updated `_build_system_message()` (bot_access.py:384)

**Current:**
```python
SECURITY_PREAMBLE + _bot_config_block() + personal_ctx + memory_note + lang_instr
```

**After:**
```python
def _build_system_message(chat_id: int, user_text: str = "") -> str:
    role_key = _get_prompt_role_key(chat_id)
    lang = _resolve_lang(chat_id, user_text)
    tpl = PROMPTS["role_system_prompts"].get(role_key, PROMPTS["role_system_prompts"]["user"])
    ctx = {
        "security_preamble": SECURITY_PREAMBLE,
        "bot_name": BOT_NAME,
        "user_name": _get_user_name(chat_id),
        "lang_instruction": PROMPTS["lang_instructions"].get(lang, PROMPTS["lang_instructions"]["en"]),
        "bot_capabilities": PROMPTS["role_capabilities"].get(role_key, ""),
        "style_guide": PROMPTS["role_styles"].get(role_key, ""),
        "calendar_summary": _calendar_context(chat_id) if role_key != "guest" else "",
        "notes_summary": _notes_context(chat_id) if role_key not in ("guest", "voice") else "",
        "contacts_summary": _contacts_context(chat_id) if role_key not in ("guest", "voice") else "",
        "memory_note": "..." if role_key != "guest" else "",
    }
    return fmt_prompt(tpl, **ctx)
```

### Admin Hot-Reload

Since templates live in `prompts.json`, hot-reload is:
```python
import importlib
import core.bot_prompts as _pm
importlib.reload(_pm)
from core.bot_prompts import PROMPTS, fmt_prompt
```
Or simpler: re-read the JSON file on admin command.
