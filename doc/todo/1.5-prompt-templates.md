## Problem

Currently all user roles receive the same system prompt structure (built in `_build_system_message()` → `src/telegram/bot_access.py:384`). The prompt is assembled from hardcoded blocks:

```python
SECURITY_PREAMBLE + _bot_config_block() + personal_ctx + memory_note + lang_instr
```

This means:
- Guests get the same tone/style/content scope as admins
- Personal context (notes, calendar, contacts) is injected for all roles — leaks privilege expectations
- Conversation style cannot differ per role (demo/onboarding vs. expert/professional)
- Admins cannot customize the system prompt without code changes

---

## Goal

Replace hardcoded prompt assembly with **role-based template files**, keeping the variable injection mechanism.

---

## Design

### Template File Structure

```
src/prompts/
├── guest_system.txt        ← System prompt for guests (shared RAG only, onboarding style)
├── user_system.txt         ← System prompt for approved users (full personal context)
├── admin_system.txt        ← System prompt for admins (same as user + admin awareness)
├── voice_system.txt        ← System prompt for voice (concise, no markdown)
├── guest_style.md          ← Conversation style guide for guests
├── user_style.md           ← Conversation style guide for users
└── admin_style.md          ← Conversation style guide for admins
```

Templates are loaded at startup and cached. Can be reloaded via admin command.

---

### Template Variables

| Variable | Description | Guest | User | Admin | Voice |
|---|---|---|---|---|---|
| `{bot_name}` | Bot name from `BOT_NAME` | ✅ | ✅ | ✅ | ✅ |
| `{user_name}` | User's display name | ✅ | ✅ | ✅ | ✅ |
| `{language_instruction}` | Language directive | ✅ | ✅ | ✅ | ✅ |
| `{security_preamble}` | Prompt injection guard | ✅ | ✅ | ✅ | ✅ |
| `{rag_context}` | Retrieved document context | shared only | full | full | full |
| `{calendar_summary}` | User's upcoming events | ❌ | ✅ | ✅ | ✅ |
| `{notes_summary}` | User's recent notes | ❌ | ✅ | ✅ | ❌ |
| `{contacts_available}` | Whether contacts loaded | ❌ | ✅ | ✅ | ❌ |
| `{memory_note}` | Long-term memory summary | ❌ | ✅ | ✅ | ❌ |
| `{style_guide}` | Injected from `{role}_style.md` | ✅ | ✅ | ✅ | ❌ |
| `{admin_context}` | Admin system status note | ❌ | ❌ | ✅ | ❌ |

---

### Role → Template Mapping

| Role | Template file | Context level |
|---|---|---|
| `guest` | `guest_system.txt` | Minimal: shared RAG + style only |
| `user` | `user_system.txt` | Full: calendar, notes, contacts, RAG |
| `advanced` | `user_system.txt` | Same as user (advanced features don't change prompt) |
| `admin` | `admin_system.txt` | User template + admin awareness note |
| `developer` | `admin_system.txt` | Same as admin |
| `voice` | `voice_system.txt` | Concise: no markdown, no lists, 2 sentences max |

---

### Example: Guest System Prompt Template

```
{security_preamble}

You are {bot_name}, a helpful AI assistant. You are speaking with a guest user named {user_name}.

{style_guide}

You have access to the following shared knowledge:
{rag_context}

Always respond in {language_instruction}.
Keep your response concise and welcoming. Guide the user toward requesting full access for more features.
```

### Example: User System Prompt Template

```
{security_preamble}

You are {bot_name}, a personal AI assistant for {user_name}.

{style_guide}

{rag_context}

{calendar_summary}

{notes_summary}

{memory_note}

Always respond in {language_instruction}.
```

## Architecture

### New Module: `src/core/prompt_loader.py`

| Function | Signature | Description |
|---|---|---|
| `load_templates()` | `() -> dict[str, str]` | Load all `src/prompts/*.txt` at startup, cache in `_templates` |
| `get_template(role)` | `(str) -> str` | Return template for role; fallback to `user_system.txt` |
| `render_template(role, ctx)` | `(str, dict) -> str` | Replace `{variable}` with values from `ctx` dict |
| `reload_templates()` | `() -> None` | Hot-reload templates without restart (admin command) |

### Hook Point: `_build_system_message()` in `src/telegram/bot_access.py`

**Current (line 384):**
```python
def _build_system_message(chat_id, lang):
    parts = [SECURITY_PREAMBLE, _bot_config_block(), personal_ctx, memory_note, lang_instr]
    return "\n\n".join(filter(None, parts))
```

**After template implementation:**
```python
def _build_system_message(chat_id, lang):
    role = _get_user_role(chat_id)           # guest / user / admin / voice
    ctx = _build_prompt_context(chat_id, lang, role)   # collects all variables
    return render_template(role, ctx)
```

## Conversation Style Documents

Each template references a `{role}_style.md` that describes the tone and style for that role:

| File | Style description |
|---|---|
| `src/prompts/guest_style.md` | Welcoming, onboarding-focused, concise, no jargon, encourages full registration |
| `src/prompts/user_style.md` | Professional, helpful, context-aware, can use markdown |
| `src/prompts/admin_style.md` | Efficient, technically precise, can discuss system state |

Style guides are injected via `{style_guide}` variable.

---

## Admin Configurability

Phase 3 (Admin UI to edit templates) — via admin panel:

- Admin → LLM Settings → 📝 System Prompts
- Shows current template content per role (read-only view in Phase 3, edit in Phase 4)
- `/admin_reload_prompts` command → calls `reload_templates()` without restart

Phase 4 (per-user custom prompt):

- Admin can set a one-off override prompt for a specific user
- Stored in user record (`custom_system_prompt` field)
- `_build_system_message()` checks for override first before role template
